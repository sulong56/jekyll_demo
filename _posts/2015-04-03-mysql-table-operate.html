2.SELECT -- 查找记录

 SELECT 表达式[,....] from 数据表参照

  [where  条件]

  [group by 分组字段]

  [having 分组条件]

  [order by 排序字段[asc|desc][,...]]

  [limit [偏移值,]记录数目]

说明:

A、星号(*)表示所有字段

B、字段 [[as]别名]

C、数据表 [[as] 别名]

D、order by,对于查找结果进行排序，其中asc(默认)为升序，desc为降序。

E、limit,限制返回的记录数,偏移值从0开始编号。

F、分页时的偏移值的计算公式：(页码-1)*每页显示的记录数

C.[not] like '表达式'

四、连接 JOIN

连接是外键约束的逆操作，可以将多张数据表的数据组合在一起来使用。

左外连接 left join

(显示左表的全部及右表中符合连接条件记录)

数据表 left [outer] join 数据表 on 连接条件

SELECT * FROM a LEFT JOIN  b  ON a.aID =b.bID 

left join是以A表的记录为基础的,A可以看成左表,B可以看成右表,left join是以左表为准的. 
换句话说,左表(A)的记录将会全部表示出来,而右表(B)只会显示符合搜索条件的记录(例子中为: A.aID = B.bID).
B表记录不足的地方均为NULL.

右外连接 right join

(显示右表的全部及左表中符合连接条件记录)

数据表 right [outer] join 数据表 on 连接条件

相等联接或内联接 ​inner join

SELECT *  FROM a,b WHERE a.aID = b.bID 

很明显,这里只显示出了 A.aID = B.bID的记录.这说明inner join并不以谁为基础,它只显示符合条件的记录.

如果在INNER JOIN操作中要联接包含Memo 数据类型或 OLE Object 数据类型数据的字段，将会发生错误。

语法汇总

​语法：FROM table1 LEFT JOIN table2 ON table1.field1 compopr table2.field2  
说明：table1, table2参数用于指定要将记录组合的表的名称。 
field1, field2参数指定被联接的字段的名称。且这些字段必须有相同的数据类型及包含相同类型的数据，但它们不需要有相同的 
名称。 
compopr参数指定关系比较运算符："="， "<"， ">"， "<="， ">=" 或 "<>"。

2.引发子查询的情况

B、由比较运算符引发的子查询

   如果子查询返回多个结果：

   all，any(some)

   > >= all --> 子查询将取最大值

   > >= any --> 子查询将取最小值

   = any --> 等价于in

   != all --> 等价于not in

   <= < all --> 子查询将取最小值

   <= < some --> 子查询将取最大值
